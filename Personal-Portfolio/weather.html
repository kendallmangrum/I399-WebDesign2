Get Weather Using JavaScript & OpenWeatherMap API

For my Web Design II class we were tasked with using JavaScript and the OpenWeatherMap API to create a webpage that takes in either an Indiana city/county name or any United States zipcode and outputs the current weather. You can interact with the webpage here or check it out on GitHub.

Input the City Name

The first option the user has to search for a certain location's current weather is using a city name. The user just needs to type in a city name using its proper capitalization and spacing, and the program with output the corresponding information for that city. This program is limited to cities within Indiana however. The user can however search county names as well if they wanted. They can also receive the temperature in either Fahrenheit or Celsius.

Input a Zipcode

The user also has the option to search the current weather using a zipcode. The zipcode feature isn't limited to only Indiana however. It can be used to search for the current weather for any zipcode within the United States. Something else to note is that the background image changes depending on the current weather for each location. If the wind speed is 35 mph or greater, an alert also pops up on screen.

City Name Error

If the user types in the name of a city wrong or if that city isn't included in the city names database file, "Invalid City Name" appears on the screen letting the user know what happened.

Zipcode Error

If the user types in an invalid zipcode, a message saying "There was a 404 error..." appears on the screen. This is because the zipcode is plugged into a url to find the data, so if the zipcode is invalid, the url will be too.
